(`npm run db:up && npm run db:apply && npm run test:db`). Identidade injetada por
conexão (`SET ROLE authenticated` + `request.jwt.claims`), fazendo `auth.uid()` e RLS
valerem de verdade — primeira vez que RLS/RPCs/triggers do CRM são executados em teste.

**Método**: os testes afirmam o comportamento CORRETO da regra de negócio. Quando o
banco divergia, o caso era marcado `it.fails` com "BUG descoberto" — e só virou `it`
depois da migration de correção correspondente. Nenhum teste foi afrouxado para passar.

Bugs reais encontrados pela suíte e corrigidos (detalhe em 2026-07-19-correcoes.md):
roubo de lead pelo motor ponderado (+ concorrência), follow-up ressuscitado em lead
morto, dedup ausente para leads sem projeto (+55, lixeira), INSERT já fechado,
escopo de gestor divergente em 6 RPCs de leitura, venda pendente contada como venda,
lead soft-deletado na aderência, perda invisível nas métricas, pos_venda como aberto,
funil perdendo status legados, lista do corretor escondendo 'novo'.
